# Code Review — 2026-03-06

## Architecture Summary
`lunduke-transcripts` is a local-first CLI pipeline centered on `lunduke-transcripts run`, which resolves YouTube and local-file targets, persists source state in SQLite, fetches captions or ASR transcripts, extracts scene-change frame candidates with `ffmpeg`, and writes per-video artifacts under `data/videos/`. The main risk areas are the external-process boundaries (`yt-dlp`, `ffmpeg`, `ffprobe`, optional ASR/LLM providers), user-supplied file paths, and the idempotency contract between discovered source IDs, stored rows, and artifact directories.

## Remediation Roadmap

### Fix Now (Blockers)
- R001 — The canonical bundle is the whole point of Sprint 7. Marking a run successful without it undermines the phase’s contract.
- R002 — Failed reprocesses should not destroy previously good frame artifacts.

### Fix Soon (High ROI)
- R003 — This will confuse real users the first time they rename or touch a local file and suddenly get a second artifact tree. Effort: M.

### Fix Later (Refactors)
- R004 — Low risk, but it keeps causing process drift and incomplete reviews.

## Patch Suggestions

For R001, make frame generation gate the processed result or preserve a degraded-but-explicit bundle status.

```python
# src/lunduke_transcripts/app/single_video_transcriber.py — BEFORE
frame_manifest_path = self._write_frame_manifest(...)
if frame_manifest_path is not None:
    tutorial_asset_bundle_path = self.storage.write_video_artifact(...)

# AFTER
frame_manifest_path = self._write_frame_manifest(...)
if frame_manifest_path is None:
    raise RuntimeError("frame_manifest_generation_failed")
tutorial_asset_bundle_path = self.storage.write_video_artifact(...)
```

For R002, stage frame extraction in a temporary directory and only replace `frames/` after success.

```python
# src/lunduke_transcripts/infra/video_frame_extractor.py — BEFORE
for existing in output_dir.glob(f"*.{self.image_format}"):
    existing.unlink()

# AFTER
tmp_dir = output_dir.parent / f".{output_dir.name}.tmp"
tmp_dir.mkdir(parents=True, exist_ok=True)
# write new frames into tmp_dir
# on success: replace output_dir with tmp_dir atomically
```
